## [2024-01-16] GPU调度器实现

### 原因
需要支持多GPU并行和负载均衡。

### 实现内容
1. GPU自动检测 (NVML)
2. 四种调度策略
3. 显存估算
4. 状态监控

### 风险

- 显存估算不准确

### 性能收益
- 多GPU利用率提升
- 任务分配更均匀

### 回滚方案
```rust
// 使用模拟模式

    gpus.push(GPUInfo {
        id: GPUId(0),
        name: "Simulated GPU".to_string(),
        total_memory_mb: 8192,
        compute_capability: (8, 6),
    });
}
```

## [2024-01-17] 低显存优化

### 原因
需要支持8GB显存的低端显卡。

### 实现内容
1. FP16 半精度
2. 自动混合精度
3. 模型分块加载
4. CPU Offload
5. 分辨率自适应
6. 批量大小自动调节

### 风险
- FP16可能影响画质
- Offload会降低速度

### 性能收益
- 8GB显存可运行SDXL
- 显存占用降低约60%

### 回滚方案
```toml
[gpu]
fp16 = false
cpu_offload = false
```

## [2024-01-18] 混合算力调度

### 原因
需要智能调度本地和云端资源。

### 实现内容
1. 执行模式选择
2. 自动降级策略
3. 负载均衡
4. 成本估算
5. 配额管理

### 风险
- 云端API不稳定
- 成本控制困难

### 性能收益
- 本地负载高时自动切换云端
- 节省云端成本

### 回滚方案
```rust
// 强制使用本地
params.execution_mode = ExecutionMode::Local;
```
